# Task #1 — CLI Calculator

## Цель

Написать консольный калькулятор на Go, который принимает два числа и оператор как аргументы командной строки и выводит результат.

**Главная цель — не калькулятор, а прокачка базовых конструкций Go в чистом виде, без фреймворков.**

---

## Acceptance Criteria

Программа считается сделанной, если выполняются все пункты:

- [ ] `./calc 10 + 5` → `15`
- [ ] `./calc 10 / 3` → `3.3333333333333335` (не `3`)
- [ ] `./calc 10 / 0` → сообщение в stderr + exit code 1
- [ ] `./calc 10 ^ 2` → сообщение в stderr + exit code 1
- [ ] `./calc 10` → сообщение в stderr + exit code 1
- [ ] `./calc abc + 5` → сообщение в stderr + exit code 1
- [ ] `go vet ./...` проходит без предупреждений
- [ ] Нет сторонних зависимостей (`go.mod` содержит только `module` и `go` директивы)

---

## Технические требования

### Обязательно

| Требование | Детали |
|---|---|
| Аргументы | Ровно 3: `<число> <оператор> <число>` через `os.Args[1:]` |
| Числа | Парсить как `float64` через `strconv.ParseFloat` |
| Операторы | `+` `-` `*` `/` |
| Ошибки | `fmt.Fprintf(os.Stderr, "ошибка: ...")` + `os.Exit(1)` |
| Структура | Весь код в одном файле `main.go` |
| Зависимости | Только стандартная библиотека |

### Запрещено

- `fmt.Println` или `fmt.Print` для вывода ошибок — только `fmt.Fprintf(os.Stderr, ...)`
- Сторонние библиотеки (`cobra`, `flag`, и т.д.)
- `panic` для обработки ошибок

---

## Темы Go, которые ты прокачиваешь

Это не просто список — это checklist того, что ты **обязан использовать** в реализации:

- **`os.Args`** — получить аргументы командной строки
- **`strconv.ParseFloat`** — конвертировать строку → `float64`, обработать ошибку
- **`switch`** — выбор операции по оператору, явный `default` для неизвестных
- **`fmt.Fprintf(os.Stderr, ...)`** — правильный вывод ошибок
- **`os.Exit(1)`** — завершение программы с кодом ошибки
- **Множественный возврат** — функции типа `func calculate(...) (float64, error)`

## Подсказки по архитектуре

Раздели логику на функции — не пиши всё в `main()`:

```
main()
  └── parseArgs(args []string) → (float64, string, float64, error)
  └── calculate(a float64, op string, b float64) → (float64, error)
```

Это не обязательное требование, но хорошая практика даже для маленьких программ.
